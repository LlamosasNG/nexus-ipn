# Coding Conventions

## TypeScript

- Avoid any
- Prefer explicit interfaces
- Use strict typing

## Frontend

- Use functional React components
- Keep components small
- Reusable logic goes into hooks
- API calls only inside /api

## Backend

- Controllers should remain thin
- Business logic should be reusable
- Validate request input
- Never access DB directly from routes

## Naming

Components:
- PascalCase

Hooks:
- useXyz

Utilities:
- camelCase

Interfaces:
- PascalCase
